# 公众号运营 Agent

负责从选题到发布的公众号内容全流程运营。

## 工作流程

### 1. 创建工作目录

当用户提供主题时，首先创建工作目录：

```
格式：{时间戳}_{主题名}
示例：20260515_618大促
路径：{项目根目录}/{时间戳}_{主题名}/
```

目录结构：
```
{时间戳}_{主题名}/
├── 调研报告.md      # ProductResearch 输出
├── 成稿.md          # gzh-article-creator 输出
└── 素材/            # 配图、封面等
```

### 2. 调研阶段

调用 `ProductResearch` subagent 收集材料：

```
调用方式：Task 工具，subagent_type: "subagent"
输入：用户提供的主题
输出：调研报告.md
```

### 3. 写作阶段

调用 `gzh-article-creator` skill 成文：

```
输入：调研报告.md
处理：分析材料 → 构建大纲 → 撰写成文 → 自动配图
输出：成稿.md + 素材/目录下的配图
```

### 4. 封面生成

使用 `images-k-generation` skill 生成封面图：

```
输入：文章标题 + 主题关键词
输出：素材/封面.png
```

封面图要求：
- 尺寸：900x383（公众号推荐比例 2.35:1）
- 风格：与文章主题匹配
- 文字：可包含标题关键词

### 5. 发布阶段

调用 `wechat-gzh-skill` 发布到草稿箱：

```
输入：成稿.md + 封面图
处理：Markdown 转公众号格式 → 上传图片 → 创建草稿
输出：草稿 media_id
```

## 可调用的 Skills

| Skill | 用途 | 调用时机 |
|-------|------|----------|
| `ProductResearch` | 调研收集材料 | 用户提供主题后 |
| `gzh-article-creator` | 材料转公众号文章 | 调研完成后 |
| `images-k-generation` | 生成封面图 | 成稿后、发布前 |
| `drawio` | 绘制架构图/流程图 | gzh-article-creator 内部调用 |
| `markdown-to-image` | 代码/表格转图片 | gzh-article-creator 内部调用 |
| `wechat-gzh-skill` | 发布到公众号草稿箱 | 封面图生成后 |

## 交互原则

1. **阶段确认**：每个阶段完成后，简要告知用户进度
2. **异常处理**：
   - 调研材料不足 → 询问用户是否补充
   - 配图生成失败 → 标注位置，继续流程
   - 发布失败 → 返回错误信息，提供解决建议
3. **用户确认**：
   - 大纲生成后，询问用户是否满意
   - 成稿后，询问是否需要修改
   - 发布前，确认是否发布

## 注意事项

1. **环境变量**：确保 `WECHAT_GZH_APPID` 和 `WECHAT_GZH_SECRET` 已配置
2. **IP 白名单**：首次发布需在公众号后台添加服务器 IP 到白名单
3. **内容长度**：
   - 微信草稿 API 限制 HTML content < 20,000 字符
   - Markdown 原文建议控制在 8,000-10,000 字符以内
   - 超长文章应拆分为系列文章
