# 公众号文章创作工具

将调研材料转换为公众号风格的文章，自动判断配图需求并生成。

## 输入要求

接收以下形式的材料：
- 调研报告 Markdown 文件（ProductResearch 输出）
- 用户提供的素材文本
- 多个相关文档

## 执行流程

### 步骤一：材料分析

1. 读取并分析输入材料
2. 提取核心观点、关键数据、典型案例
3. 识别文章主题和目标读者
4. 判断文章类型：
   - 热点解读型（时效性强，观点鲜明）
   - 经验分享型（个人经历，实用性强）
   - 趋势分析型（行业洞察，深度为主）
   - 教程干货型（操作步骤，清晰为主）

### 步骤二：构建大纲

根据文章类型选择结构模板：

**热点解读型**：
```
开头：热点事件引入，制造话题感
展开：事件背景 → 各方观点 → 核心争议
分析：深层原因 → 影响范围 → 未来走向
结尾：观点升华 → 互动引导
```

**经验分享型**：
```
开头：个人故事/场景切入
展开：遇到的问题 → 解决过程
分析：关键收获 → 可复用的经验
结尾：情感升华 → 读者互动
```

**趋势分析型**：
```
开头：行业变化/数据引入
展开：现状描述 → 变化信号
分析：驱动因素 → 影响维度 → 未来预判
结尾：行动建议 → 开放讨论
```

**教程干货型**：
```
开头：痛点场景/需求引入
展开：解决方案概述
分析：详细步骤 → 注意事项 → 常见问题
结尾：总结回顾 → 扩展资源
```

### 步骤三：撰写成文

**写作规范**：

1. **开头技巧**
   - 热点引入：引用名人观点/热门事件，制造话题感
   - 故事引入：从日常场景切入，真实感强
   - 数据引入：用具体数字制造冲击力
   - 开头长度：3-5 行，快速入题

2. **段落节奏**
   - 每段 2-4 行，手机阅读友好
   - 长句拆分，避免大段文字
   - 段落间用空行分隔

3. **金句处理**
   - 关键观点用 **粗体** 突出
   - 核心结论独立成段
   - 引用内容用引用格式

4. **语言风格**
   - 口语化表达，避免书面腔
   - 适当使用"其实""换句话说""说白了"等连接词
   - 技术名词首次出现时简要解释
   - 可适当使用网络用语，但不过度

5. **避免的问题**
   - ❌ "综上所述"等正确的废话
   - ❌ "双刃剑"等陈词滥调
   - ❌ 大段专业术语堆砌
   - ❌ 过度使用"不仅...而且"句式

### 步骤四：配图决策

**自动判断配图需求**：

1. **必须配图的情况**
   - 文章涉及产品/工具介绍 → 生成产品截图或示意图
   - 概念关系复杂 → 生成架构图/流程图
   - 数据对比 → 生成对比图表
   - 步骤说明 → 生成步骤示意图

2. **配图位置**
   - 章节开头：帮助读者快速理解即将讨论的内容
   - 关键概念处：视觉化抽象概念
   - 对比分析处：增强对比效果

3. **配图生成方式**
   - 架构图/流程图 → 调用可用的图表/绘图 skill
   - 产品图/示意图 → 调用 `images-k-generation` skill
   - 代码/表格 → 调用 `markdown-to-image` skill

4. **配图嵌入**
   - 生成图片保存到 `素材/` 目录
   - 在 Markdown 中使用相对路径引用
   - 添加图片说明（斜体格式）

### 步骤五：输出文件

1. 生成最终 Markdown 文件
2. 写入路径：`{工作目录}/成稿.md`
3. 返回文件路径和文章摘要

## 输出格式

纯 Markdown 文件，结构如下：

```markdown
# {文章标题}

{开头段落}

## {章节一标题}

{章节内容}

![{图片描述}](素材/图片名.png)
*图：{图片说明}*

## {章节二标题}

{章节内容}

---

{结尾段落}
```

## 使用示例

**输入**：调研报告 `20260515_618大促/调研报告.md`

**输出**：
- `20260515_618大促/成稿.md`
- `20260515_618大促/素材/配图1.png`
- `20260515_618大促/素材/配图2.png`

## 字符长度限制

**重要**：微信公众号草稿 API 限制 HTML 内容 < 20,000 字符。

Markdown 转换为 HTML 后，字符数会显著增加（约 1.5-2 倍）。因此：

1. **成稿字数控制**：Markdown 原文建议控制在 **8,000-10,000 字符**以内
2. **超长处理**：
   - 材料过多时，优先提取核心内容，精简成文
   - 若预估会超限，主动询问用户是否拆分为系列文章
3. **精简技巧**：
   - 删除冗余表述，保留核心观点
   - 复杂表格转为图片（调用 `markdown-to-image`）
   - 长代码块转为图片或删除

## 注意事项

1. **材料不足时**：主动询问用户补充信息
2. **主题不明确时**：先与用户确认文章定位
3. **配图生成失败时**：在文中标注 `[需配图]`，继续成文
4. **保持原文观点**：不歪曲材料原意，只做风格转换
5. **字符超限时**：提醒用户文章过长，建议拆分或精简
