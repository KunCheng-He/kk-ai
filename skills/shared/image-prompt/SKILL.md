# AI Image Prompt Generator

为 AI 图像生成模型生成高质量、结构化的提示词。本 skill 仅生成提示词文本，不调用任何 API。

## 核心原则

AI 图像生成的输出质量高度依赖提示词的**细节丰富度**和**结构化程度**。一条好的提示词像电影镜头描述——画面构成、光线、色彩、纹理、文字内容都需要明确指定。

**最重要的原则：根据用户需求的特异性决定增强程度，而非盲目套用模板。**

## 支持的宽高比

提示词中用自然语言指定比例，如 "16:9 宽高比" 或 "超宽全景 8:1"。

| 比例 | 适用场景 |
|------|----------|
| 1:1 | 头像、产品图、社交媒体方图 |
| 16:9 | 横版海报、网页横幅 |
| 9:16 | 竖版海报、手机壁纸 |
| 21:9 | 宽屏电影感画面 |
| 4:1 / 1:4 | 特写长条、竖版长图 |
| 8:1 / 1:8 | 全景横卷、超长竖版 |

## 生成流程

### Step 1: 分析需求 + 意图识别

判断用户意图类型和特异性：

| 意图类型 | 说明 | 后续步骤 |
|----------|------|----------|
| **从零生成** | 用户提供粗略想法，从头构建提示词 | 进入 Step 2 |
| **精修优化** | 用户已有草稿，针对性强化而非推翻重写 | 进入 Step 2 |
| **编辑修图** | 用户想修改现有图片（修图、换背景、局部修改、风格迁移） | 进入 [编辑流程](#编辑流程) |
| **模糊需求** | 用户需求不清 | 主动询问三个核心问题——**主体内容？风格偏好？用途？** |

### Step 2: 选择增强策略并阅读参考

根据需求特异性选择策略：

| 特异性 | 策略 | 参考文件 |
|--------|------|----------|
| **详细具体** | 保留用户需求，仅结构化整理 | 无需额外参考 |
| **比较泛化** | 依据 `codex-prompting.md` 的增强规则 + 从场景模板中提取补充细节 | `codex-prompting.md` + 对应场景参考文件 |
| **非常模糊** | 停止生成，回到 Step 1 主动追问主体、风格、用途 | — |

**判断信号：**
- 详细具体：已包含主体描述、风格要求、构图说明、颜色等具体细节
- 比较泛化：有大致方向但缺少关键维度（如"画一张好看的风景图"）
- 非常模糊：几乎无方向（如"帮我画张图"）

**文件加载原则**：优先加载匹配场景的参考文件；跨场景请求时加载多个相关参考取交集（如"动漫风格的产品海报"需同时参考 `anime.md` 和 `poster.md`）。`codex-samples.md` 可在任何场景下查阅同类提示词的结构参考。

**场景参考路由**（泛化时从匹配模板中提取构图/灯光/风格关键词）：

| 场景 | 参考文件 | 提示词中可提取的元素 |
|------|----------|---------------------|
| 海报/广告 | `references/poster.md` | 排版布局、视觉层次、色彩搭配 |
| 产品/商业摄影 | `references/commercial-photo.md` | 布光方式、材质表现、产品摆放 |
| 人物写真 | `references/portrait.md` | 姿态表情、光线方向、皮肤质感 |
| 动漫/二次元 | `references/anime.md` | 画风特征、色彩倾向、线条风格 |
| 全景/超长图 | `references/panorama.md` | 比例选择、元素分布、视野控制 |
| 故事板 | `references/storyboard.md` | 分镜节奏、角色一致性、场景连接 |
| 信息图/食谱 | `references/infographic.md` | 数据呈现、步骤排列、视觉引导 |
| 窗外风景 | `references/window-view.md` | 透视关系、景深处理、季节表达 |
| 城市地标 | `references/city-landmark.md` | 地标特征、光影时间、人文氛围 |
| 文创产品 | `references/creative-product.md` | 产品质感、摆放方式、包装风格 |
| 创意玩法 | `references/creative-play.md` | 视觉效果、混合手法、风格融合 |
| 通用风格 | `references/style-keywords.md` | 风格关键词速查、画质术语 |

### Step 3: 输出提示词

**默认输出中文提示词**。仅在以下情况输出英文：
- 用户明确要求英文提示词
- 全球化翻译场景（参考 `poster.md` 翻译模板）
- `window-view.md` 场景（英文提示词效果更好）
- 图片中文字需要翻译为其他语言时（英文提示词 + 翻译指令）

输出内容必须包含：

1. **最终提示词**：代码块包裹，可直接复制使用
2. **设计思路**：1-2 句话说明选择此构图/光线/风格的理由
3. **使用建议**：推荐宽高比、注意事项
4. **变体建议**：1-2 个风格或构图的变体思路（可选）

### Step 4: 保存（如需要）

如果用户要求保存到指定目录，写入文件，命名格式：`{主题关键词}-{YYYYMMDD}.md`

## 编辑流程

当用户意图为编辑修图时，参考 `codex-prompting.md` § Edit use-case tips 和 `codex-samples.md` § Edit 模板：

1. **确认编辑范围**：明确哪些部分要修改、哪些要保持不变
2. **标注参考图**：按索引和角色标注图片（`图1：编辑目标`、`图2：风格参考`）
3. **使用约束语言**：`仅修改 X；保持 Y 不变`，并在每次迭代中重复关键约束
4. **选择编辑模板**：从以下类型中匹配：

| 编辑类型 | 说明 |
|----------|------|
| text-localization | 仅修改文字；保留布局、排版、间距和层级 |
| identity-preserve | 锁定身份（面部、身体、姿势、发型、表情）；仅修改指定元素 |
| precise-object-edit | 精确指定移除/替换内容；保留周围质感和光照 |
| lighting-weather | 仅修改环境条件（光照、阴影、氛围）；保持几何和主体身份 |
| background-extraction | 纯色平面背景抠图；清晰轮廓；充足留白；无阴影无光晕 |
| style-transfer | 指定保留的风格线索（调色板、质感、笔触）和要修改的内容 |
| compositing | 按索引引用素材；指定移动内容和位置；匹配光照、透视和比例 |
| sketch-to-render | 保留布局、比例和透视；选择支持草图的材质和光照 |

## 提示词结构

无论采用哪种策略，最终提示词应使用以下结构（按需选取字段，不需要全部使用）：

```
[Use case / 用途分类] + [Asset type / 素材类型] + [Primary request / 主体描述] + [Scene/backdrop / 场景背景] + [Subject / 主体细节] + [Style/medium / 风格介质] + [Composition/framing / 构图取景] + [Lighting/mood / 光线氛围] + [Color palette / 色调] + [Materials/textures / 材质纹理] + [Text (verbatim) / 精确文字] + [Constraints / 约束] + [Avoid / 排除项]
```

- **主体描述**（必需）：人物外貌/物体形态/场景特征的详细描述
- **场景/背景**（推荐）：环境、光线、氛围
- **风格指令**（推荐）：整体视觉风格关键词，见 `references/style-keywords.md`
- **技术参数**（可选）：画质指令如 `超详细`、`8K`、`景深控制精准`
- **文字/排版要求**（按需）：明确指定文字内容、位置、字体风格
- **约束/排除项**（推荐）：明确禁止不想要的元素，可避免常见问题

## 文字渲染要点

部分 AI 图像生成模型对英文字体的渲染样式更丰富（粗体、手写体、装饰体等都有多种变体），而中文通常渲染为统一字体、缺少风格变化。因此：

1. **默认输出中文提示词**，让图片中的中文文字渲染更贴合用户语言
2. **明确指定文字内容**——不要只说"加上标题"，写明"标题文字为'星际穿越'"
3. **描述字体风格**——如"粗体无衬线字体"、"手写体"
4. **描述文字位置**——如"顶部居中"、"图片底部横条"
5. **全球化翻译场景例外**——需要将图中文本翻译为其他语言时，用英文提示词 + 翻译指令效果最好。参考 `references/poster.md` 的翻译模板
6. **模型差异提示**——如果用户指定了具体模型，可根据该模型的文字渲染特点调整提示词描述

## 角色一致性提示

在支持角色一致性的模型上，可通过以下方式保持多角色一致性：

1. 首次生成时详细描述每个角色的固定特征
2. 后续生成复用相同角色描述关键词
3. 可上传参考图片以保持一致
