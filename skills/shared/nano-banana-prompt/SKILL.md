# Nano Banana Prompt Generator

为 Google Gemini 的 Nano Banana 图像生成模型生成高质量、结构化的提示词。本 skill 仅生成提示词文本，不调用 API。

## 核心原则

Nano Banana 的生成质量高度依赖提示词的**细节丰富度**和**结构化程度**。一条好的提示词像电影镜头描述——画面构成、光线、色彩、纹理、文字内容都需要明确指定。

## 支持的宽高比

提示词中用自然语言指定比例，如 "16:9 宽高比" 或 "超宽全景 8:1"。

| 比例 | 适用场景 |
|------|----------|
| 1:1 | 头像、产品图、社交媒体方图 |
| 16:9 | 横版海报、网页横幅 |
| 9:16 | 竖版海报、手机壁纸 |
| 21:9 | 宽屏电影感画面 |
| 4:1 / 1:4 | 特写长条、竖版长图 |
| 8:1 / 1:8 | 全景横卷、超长竖版 |

## 提示词结构

一条完整的提示词由以下模块灵活组合：

```
[主体描述] + [场景/背景] + [风格指令] + [技术参数] + [文字/排版要求] + [排除项]
```

- **主体描述**（必需）：人物外貌/物体形态/场景特征的详细描述
- **场景/背景**（推荐）：环境、光线、氛围
- **风格指令**（推荐）：整体视觉风格关键词，见 `references/style-keywords.md`
- **技术参数**（可选）：画质指令如 `超详细`、`8K`、`景深控制精准`
- **文字/排版要求**（按需）：明确指定文字内容、位置、字体风格（英文渲染效果通常优于中文）
- **排除项**（可选）：避免不想要的元素

## 生成流程

### Step 1: 分析需求

判断用户意图是以下哪种：

- **从零生成**：用户提供了粗略想法，需要从头构建提示词。提取核心主体、风格偏好、文字需求、尺寸偏好、用途。
- **精修优化**：用户已有提示词草稿，需要改进。分析现有提示词的不足（细节缺失、结构混乱、风格不够具体等），针对性强化，而非推翻重写。

如果需求模糊（如"画一张好看的图"），主动询问：主体内容？风格偏好？用途？

### Step 2: 选择模板并阅读参考

根据场景类型，阅读对应的参考文件获取模板和示例：

| 用户意图 | 场景 | 参考文件 |
|----------|------|----------|
| 海报/广告设计 | 海报信息图 | `references/poster.md` |
| 产品/商业摄影 | 商业摄影 | `references/commercial-photo.md` |
| 人物写真/人像 | 人物写真 | `references/portrait.md` |
| 动漫/二次元 | 动漫风格 | `references/anime.md` |
| 全景/超长图 | 全景长卷 | `references/panorama.md` |
| 故事板/连续画面 | 故事板 | `references/storyboard.md` |
| 食谱/科普/决策图 | 信息图 | `references/infographic.md` |
| 城市窗外/天气卡片 | 窗外风景 | `references/window-view.md` |
| 城市/地标宣传 | 城市地标 | `references/city-landmark.md` |
| 冰箱贴/邮票/游戏盒 | 文创产品 | `references/creative-product.md` |
| 撕纸/次元穿越/变真² | 创意玩法 | `references/creative-play.md` |
| 风格关键词参考 | 通用 | `references/style-keywords.md` |

只需阅读匹配场景的文件，无需全部加载。

### Step 3: 输出提示词

输出内容必须包含：

1. **最终提示词**：代码块包裹，可直接复制到 Gemini 使用
2. **设计思路**：1-2 句话说明提示词的设计逻辑
3. **使用建议**：推荐宽高比、注意事项
4. **变体建议**：1-2 个风格或构图的变体思路（可选）

### Step 4: 保存（如需要）

如果用户要求保存到指定目录，写入文件，命名格式：`{主题关键词}-{YYYYMMDD}.md`

## 文字渲染要点

Nano Banana 对英文字体的渲染样式更丰富（粗体、手写体、装饰体等都有多种变体），而中文通常只能渲染为统一字体，缺少风格变化。所以：

1. **提示词本身可以用英文描述**——让模型更准确理解构图和风格意图
2. **但图片中需要显示的文字应优先使用中文**——因为用户是中文用户，图片中的文字内容应贴合用户语言。如果中文渲染效果不理想，再尝试转换为英文提示词描述中文文字内容
3. **明确指定文字内容**——不要只说"加上标题"，写明"标题文字为'星际穿越'"
4. **描述字体风格**——如"粗体无衬线字体"、"手写体"
5. **描述文字位置**——如"顶部居中"、"图片底部横条"
6. **全球化翻译场景例外**——需要将图中文本翻译为其他语言时，用英文提示词 + 翻译指令效果最好

## 角色一致性提示

Nano Banana 2 支持最多 5 个角色一致性：
1. 首次生成时详细描述每个角色的固定特征
2. 后续生成复用相同角色描述关键词
3. 可上传参考图片以保持一致
