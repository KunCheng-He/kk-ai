# 构建结构技能 (Architect Structure)

## 1. 核心定位

此技能负责知识的**中场组织**：将零散的"原子卡片"组装成有逻辑深度的"分子结构"，为后续的写作打下骨架。

## 2. 前置条件（启动时检查）

启动本技能前，先做以下检查。任一条件不满足时，按"降级策略"处理，不要硬跑流程。

| 条件 | 检查方式 | 降级策略 |
|---|---|---|
| 项目目录 `./06-AI共建/[项目名称]/知识卡片/` 存在 | `glob "06-AI共建/[项目名称]/知识卡片/*.md"` | 若不存在 → 提示"请先调用 `ob_capture_insight` 生成知识卡片，或确认项目名是否正确" |
| 项目内至少 3 张知识卡片 | glob 后计数 | **< 3 张** → 提示"素材不足，建议先用 `ob_capture_insight` 补充；如果你确认要在少量素材下强行涌现，请显式说明" |
| 知识卡片内容已可读 | 读取每张卡片的前 50 行 | 若某张卡片近乎空内容 → 提示用户补全后重试 |
| `assets/skeleton_template.md` 和 `references/thinking_models.md` 可读取 | 读取这两个文件 | 若缺失 → 提示"模板资产缺失，请检查 skill 安装完整性"，不要凭空生成 |
| 用户已确认要进入"结构涌现"阶段 | 上下文（用户在用本 skill 即视为确认） | 若用户在调用时还在发散阶段（如刚提了个新概念），建议先调 `ob_capture_insight` |

**禁止**：
- 在 < 3 张卡片的素材下默认强行涌现
- 跨项目混用素材——素材扫描范围限定在 `./06-AI共建/[项目名称]/知识卡片/` 内，不跨项目目录

## 3. 执行流程

### 步骤一：资产扫描 (Scan)
*   **动作**：
    1.  确定当前项目路径：`./06-AI共建/[项目名称]/知识卡片/`。
    2.  调用 `glob` 工具获取文件列表。
    3.  调用 `read` 工具（批量）读取卡片内容。
    4.  **异常处理**：若目录不存在，提示用户先创建卡片或指定正确路径。
    5.  **思考**：提取每张卡片的核心论点（Claim）和关键证据（Evidence）。

### 步骤二：结构涌现 (Emergence)
*   **动作**：
    1.  加载 `references/thinking_models.md` 获取灵感。
    2.  寻找卡片间的隐性连接：
        *   **因果链**：卡片A是卡片B的原因。
        *   **演变链**：卡片A是过去，卡片B是现在。
        *   **冲突链**：卡片A与卡片B观点相反，卡片C是综合。
    3.  **输出建议**：向用户提供 2-3 种可能的组织结构（例如："方案A：按历史演变；方案B：按底层原理"）。

### 步骤三：结构化产出 (Output)
*   **触发条件**：用户选择了一种结构方案。
*   **动作**：
    1.  **加载模板**：读取 `assets/skeleton_template.md`。
    2.  生成**输出卡片**。
    3.  **严格约束**：内容必须使用**逻辑骨架**形式，严禁直接生成长文段落。
         *   ✅ 正确格式：`【 核心观点 → 支撑案例(卡片A) → 理论依据(卡片B) → 延伸推论 】`
         *   ❌ 错误格式：`本文首先讨论了...其次...`
     4.  **过渡钩子**：为每个模块写一句逻辑桥梁——回答"从模块 N 到模块 N+1，认知上需要跨过什么"。用问句形式，例如：
         *   ✅ `既然 A 定义了什么，那 B 解决的是否就是"如何"？`
         *   ✅ `A 解释了"是什么"，B 该解释"为什么是现在"了`
         *   ❌ `接下来我们看下一个模块`（空信息，没有逻辑桥梁价值）
     5.  **清理模板残留**：填充完成后，删除 `assets/skeleton_template.md` 中所有 `<!--  -->` HTML 注释块（包括"模板使用说明"和"按需复制模块"两个块），不要让注释残留在输出卡片里。
     6.  **输出提示**：告知用户本骨架将被 `ob_refine_card` 消费，生成最终的 `LOOP-xxx` 卡片后，本文件保留作为结构参考。

## 4. 交互示例
> User: 这些关于 RAG 的卡片太乱了，帮我理理。
> Agent (扫描卡片后): 我读了这 5 张卡片。既然你想讲透 RAG，建议用“问题-方案”结构：
>   1. 先用“幻觉问题”卡片引入痛点。
>   2. 再用“外挂知识库”卡片解释原理。
>   3. 最后用“GraphRAG”卡片展示未来。
> User: 就按这个来。
> Agent: (生成 `结构-问题方案版.md`) 已生成结构大纲，采用了“痛点-原理-未来”的三段式逻辑。
