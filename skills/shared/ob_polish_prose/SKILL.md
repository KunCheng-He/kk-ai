# 润色文字技能 (Polish Prose)

## 1. 核心定位
此技能负责知识的**肉体塑造**：将干瘪的"逻辑骨架"填充为血肉丰满、语言生动的"正文段落"。

## 2. 执行流程

### 步骤一：加载约束 (Load Context)
*   **动作**：
    1.  读取指定的"输出卡片"或用户提供的文本。
    2.  加载 `assets/style_guide.md`，将风格约束注入上下文。
    3.  **异常处理**：若文件不存在，提示用户确认路径或直接使用默认风格。

### 步骤二：逻辑扩写 (Expand)
*   **核心原则**：**忠实于骨架，自由予表达**。
    *   **忠实**：严禁随意更改骨架中的逻辑推导链条（A -> B -> C）。
    *   **自由**：在表达方式上，使用类比、隐喻、具体的行业黑话来增强可读性。
*   **风格注入**：扩写时同步应用 `style_guide.md` 约束：
    *   章节编号用 `# N.` 格式，不用"第一幕/第二幕"
    *   每 2-3 段插入 callout 框（NOTE / TIP / QUESTION / WARNING）
    *   需要对比的内容用表格呈现，表头加 emoji
    *   首次出现的关键术语加 `**粗体**`
    *   使用第一人称"我"分享观点和判断
*   **输入示例**：`【AI降低门槛 → 垃圾内容泛滥 → 筛选成本上升】`
*   **输出示例**：
    > "AI 的确把创作门槛降到了地板上，但代价是互联网瞬间被生成的废话淹没。于是，原本用于创作的精力，现在被迫转移到了'从垃圾堆里找金子'上——筛选机制变得前所未有的昂贵。"

### 步骤三：风格审计 (Style Audit)
*   **核心**：基于 `assets/style_guide.md` 第十三节的自查清单，逐条审计输出文本。
*   **必须通过的检查项**：
    1.  标题有 emoji 且口语化？→ 若无 emoji，补上
    2.  开头是"本文将..."吗？→ 若是，用比喻/场景/反问重写
    3.  有"第一幕/第二幕"等剧本式编号吗？→ 改为 `# N.` 数字编号
    4.  有"综上所述""双刃剑""不仅...而且""随着...的发展"吗？→ 全部删除或改写
    5.  callout 框够多吗？→ 至少 3-5 个/千字，不够就加
    6.  该用表格对比的地方用了吗？→ 方案对比、优劣对比必须用表格
    7.  代码块是否指定语言、是否可运行？→ 补全语言标签，确保不是伪代码
    8.  结尾是"希望本文对你有帮助"吗？→ 若是，用个人判断/回扣比喻重写
    9.  读起来像论文还是像聊天？→ 翻译腔改口语，长句拆短句
    10. 是否有"我"的个人经验和判断？→ 若无，加上至少一处第一人称观点

### 步骤四：更新交付 (Update)
*   **动作**：
    *   直接在原文件中更新内容，或者在对话框中输出供用户确认。

## 3. 交互示例
> User: 帮我润色这段关于 Agent 的描述。
> Agent (加载 Style Guide): 现在的版本有点像说明书。我试着改了一下，用"外挂大脑"这个比喻来贯穿全文，去掉了"不仅...而且"句式，你看看这个版本感觉如何？
