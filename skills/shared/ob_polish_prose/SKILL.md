# 润色文字技能 (Polish Prose)

## 1. 核心定位

此技能负责知识的**肉体塑造**：将干瘪的"逻辑骨架"填充为血肉丰满、语言生动的"正文段落"。

**最重要的原则**：你不是在写技术博客，你是在替 hkc 写他的个人笔记。先成为他，再动笔。

## 2. 前置条件（启动时检查）

启动本技能前，先做以下检查。任一条件不满足时，按"降级策略"处理，不要硬跑流程。

| 条件 | 检查方式 | 降级策略 |
|---|---|---|
| `assets/author_voice.md` 可读取 | 读取该文件 | **必须**：若缺失，提示"人格资产是核心约束，缺它写不出 hkc 的声音"，不要用默认风格敷衍 |
| `assets/style_guide.md` 可读取 | 读取该文件 | **必须**：若缺失，提示"风格手册是核心约束，缺它无法审计"，不要用默认风格敷衍 |
| 用户已指定输入 | 上下文是否有"输出卡片"路径 / 待润色草稿 | 若无 → 询问用户：是润色一张已生成的"输出卡片"，还是直接给一段草稿 |
| 骨架来源可追溯（如果是润色输出卡片） | 检查输出卡片是否有 `source_cards` 字段 | 若无 → 仍可润色，但提示"该输出卡片缺失素材溯源，润色时无法核对事实边界" |
| 风格审计清单来源明确 | 引用 `style_guide.md` 第十三节 | 不允许在 SKILL 中维护独立清单副本——已经统一到 style_guide.md |

**禁止**：
- 跳过 author_voice/style_guide 加载
- 在没有骨架的情况下凭空扩写（应先调用 `ob_architect_structure`）
- 把"我"加进 AI 不知道 hkc 是否真说过的内容（这是诚实层的失败）

## 3. 执行流程

### 步骤一：加载人格与约束 (Load Persona & Context)
*   **动作**：
    1.  **先加载 `assets/author_voice.md`** — 这是人格注入层。读完后，用 hkc 的视角思考。
    2.  **再加载 `assets/style_guide.md`** — 这是格式规范层。
    3.  读取指定的"输出卡片"或用户提供的文本。
    4.  **异常处理**：若文件不存在，提示用户确认路径或直接使用默认风格。

### 步骤二：逻辑扩写 (Expand)
*   **核心原则**：**忠实于骨架，表达自由**。
    *   **忠实**：严禁随意更改骨架中的逻辑推导链条（A -> B -> C）。
    *   **自由**：在表达方式上，可以使用类比、隐喻，但遵守 author_voice 的克制原则——只在核心概念处用一个比喻，不要每个子概念都配一个。
*   **风格注入**：扩写时同步应用 style_guide 约束：
    *   章节编号用 `# N.` 格式，不用"第一幕/第二幕"
    *   每 2-3 段插入 callout 框（NOTE / TIP / QUESTION / WARNING），但不要为凑数塞空信息量的 callout
    *   需要对比的内容用表格呈现，表头加 emoji
    *   首次出现的关键术语加 `**粗体**`
    *   **强制使用第一人称"我"分享观点和判断** — 参考 author_voice.md 三板斧
    *   **诚实约束**：没实操过的功能禁止写得像用过一样。不确定的数据不准写确定数字。参考骨架素材中已有的信息，不要凭空扩展超出知识卡片范围的内容
    *   **不要机械使用"过渡钩子"**：骨架中的过渡钩子是给 AI 看的结构提示，不要在正文中照搬为过渡句
*   **输入示例**：`【AI降低门槛 → 垃圾内容泛滥 → 筛选成本上升】`
*   **输出示例**：
    > "AI 的确把创作门槛降到了地板上，但代价是互联网瞬间被生成的废话淹没。于是，原本用于创作的精力，现在被迫转移到了'从垃圾堆里找金子'上——筛选机制变得前所未有的昂贵。"

### 步骤三：风格审计 (Style Audit)
*   **核心**：加载并严格按 `assets/style_guide.md` 第十三节执行。该节是风格审计清单的**唯一权威源**（不再在本文件保留副本，避免失同步）。
*   **执行流程**：
    1.  重新阅读 `assets/style_guide.md` 第十三节（13.1-13.5）
    2.  跑完 13.1-13.4 全部可执行检查
    3.  把不通过项分类到"局部可修"和"需重写"（参见 13.5）
    4.  局部可修 → 改对应段落；需重写 → 整篇推倒

### 步骤四：更新交付 (Update)
*   **动作**：
    *   直接在原文件中更新内容，或者在对话框中输出供用户确认。

## 4. 交互示例
> User: 帮我润色这段关于 Agent 的描述。
> Agent (加载 Author Voice + Style Guide): 现在的版本不像你——没有"我"的判断。我试着重写了，加了两处你可能会有的实操感受，去掉了端水句式。你看看这个版本有没有你的味道？
