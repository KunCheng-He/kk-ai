# 写作风格指南 (Style Guide)

## 🎯 核心 Vibe
**"大话"风格，极客精神，表情符号艺术家。**
把复杂的技术原理，讲得像在和朋友吃火锅时聊的天一样通俗、热辣、爽快。
核心理念：**能用比喻绝不堆砌术语，能用 Emoji 绝不全是文字。**

## 🎨 视觉与结构规范 (Visual Identity)

1.  **Emoji 锚点系统**：
    *   不要只用纯文本标题。
    *   **强制**在二级标题(H2)和关键列表项前添加 Emoji。
    *   *常用映射*：
        *   核心原理/思考 -> 🧠 / 💡 / 💭
        *   痛点/问题 -> 💥 / 🤔 / ❓ / [?]
        *   解决方案/行动 -> ✅ / 🛠️ / 🚀 / [n]
        *   警告/注意 -> ⚠️ / 🚫
        *   底层/基础 -> 🏗️ / 🧱 / 🌲
        *   外部链接/工具 -> 🔗 / 🔧
2.  **Callout 开篇**：
    *   文章开头习惯用 Obsidian 的 `> [!question]` 或 `> [!NOTE]` 块。
    *   先抛出痛点或核心定义，迅速抓住注意力。
    *   *例*：`> [!question] 为什么我们需要...`

## 🗣️ 语态与措辞 (Voice & Tone)

1.  **拒绝“不说人话”**：
    *   ❌ “该协议旨在实现数据的加密传输。”
    *   ✅ “这就相当于把信装进了保险箱，邮递员（GFW）根本不知道里面写了啥。”
2.  **拟人化与口语化**：
    *   让技术组件“活”过来。
    *   *例*：“Chrome 也很懵”、“LLM 就像个被困在聊天框里的缸中之脑”。
3.  **第一人称代入**：
    *   多用“我们”、“你”。
    *   *例*：“想象你正在...”、“我们就一个目标...”

## 🚫 负面清单 (Ban List)

1.  **严禁**：教科书式的“综上所述”、“总而言之”、“首先/其次/最后”。
2.  **严禁**：没有感情色彩的定义堆砌（除非是数学公式推导）。
3.  **严禁**：过分严肃的学术腔。
4.  **严禁**：翻译腔（如“它允许用户去...” -> 改为“你可以...”）。

## ✅ 推荐清单 (Do List)

1.  **善用比喻**：用生活中的例子（外卖、快递、USB插口、厨房）去解释抽象技术。
2.  **金句 Punchline**：段落结尾来一句简短有力的总结。
3.  **数学与直觉并重**：如果是技术类文章，先讲直觉（Intuition），再上公式（Math），最后讲代码（Code）。
