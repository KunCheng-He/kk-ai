# 精炼卡片技能 (Refine Card)

## 1. 核心定位

此技能负责从"逻辑骨架"到"最终知识卡片"的**全流程**：先生成框架让用户确认关键判断，再填充完整内容，最后做交叉引用审计并写入文件。

**最重要的原则**：用户决定结构，AI 填充信息。目标是一张**扫一眼能找到东西的 Obsidian 知识卡片**，不是一篇叙事文章。

## 2. 前置条件（启动时检查）

| 条件 | 检查方式 | 降级策略 |
|---|---|---|
| `assets/author_voice.md` 可读取 | 读取该文件 | **必须**：若缺失，提示"人格资产是核心约束" |
| `assets/style_guide.md` 可读取 | 读取该文件 | **必须**：若缺失，提示"风格手册是核心约束" |
| `assets/output_card_template.md` 可读取 | 读取该文件 | **必须**：若缺失，提示"卡片模板资产缺失，请检查 skill 安装完整性" |
| 用户已指定输入 | 上下文是否有知识骨架文件 | 若无 → 询问用户：提供骨架路径 |
| 风格审计清单来源明确 | 引用 `assets/style_guide.md` 第十三节 | 不允许在 SKILL 中维护独立清单副本 |

**禁止**：
- 跳过约束加载
- 在确认通道中直接出完整卡片（快速通道除外）
- 把知识卡片写成叙事文章——不需要 Hook、不需要 Echo、不需要段落间过渡
- 把"我"加进 AI 不知道 hkc 是否真说过的内容

## 3. 执行流程

### 步骤一：加载约束与输入 (Load)

1. **先加载 `assets/author_voice.md`** — 人格注入层
2. **再加载 `assets/style_guide.md`** — 格式规范层
3. **加载 `assets/output_card_template.md`** — 卡片结构模板
4. 读取知识骨架文件和项目下的所有知识卡片
5. 扫描交叉引用：仅在**骨架中明确提到**的已有 KNOW-/TECH- 卡片路径上验证链接（不要 glob 整个 `01/02/03` 目录）

### 步骤二：判断路径 — 快速通道 vs 确认通道

**首先判断用户意图**：

| 用户说的 | 走哪条路 | 说明 |
|---------|---------|------|
| "直接出完整卡片" / "照这个骨架写" / "不用确认，直接写" | **快速通道** → 跳过步骤三，直接跳到步骤四（填充内容） | 用户信任骨架，不需要框架确认环节 |
| "出卡片"（未加"直接"） / "帮我精炼" / 隐式触发 | **确认通道** → 走步骤三（生成框架，等确认） | 默认走确认，确保结构对齐 |

### 步骤三：生成框架 (Frame) — 仅确认通道

基于骨架的逻辑结构，生成一个**知识卡片框架**。这个框架包含：

**必须有的**：
- Frontmatter（id 留 `[待分配]`，tags 根据骨架推断填充）
- 章节结构（`# N.` 编号 + 标题，来自骨架模块）
- 表格的表头（哪些维度需要对比，表头配 emoji）
- Callout 框的类型（NOTE/TIP/WARNING/QUESTION）和**提示文字**
- 交叉引用的占位（标 `[[待确认: 概念名]]`）
- 预估的篇幅分布（哪部分是重点，哪部分是附录）

**不写的**：
- 表格里的具体内容（除非信息可以直接从知识卡片提取且没有歧义）
- Callout 框中的判断性文字
- 大段描述性段落

**给用户的提示**：在需要用户下判断的地方，用 `<!-- TODO: 你的判断 — ... -->` 标注，例如：
```
> [!NOTE] 核心定义
> <!-- TODO: 你的判断 — 用一句话说清楚这个概念对你意味着什么 -->
```

**输出并等待**：将框架输出给用户，询问：
> 这是卡片框架。请你：
> 1. 确认章节结构是否符合你的预期
> 2. 在标注了 `<!-- TODO -->` 的地方填入你的个人判断
> 3. 告诉我还需要调整什么

**禁止在此阶段直接出完整卡片。** 必须等待用户反馈。

### 步骤四：填充内容 (Fill) — 基于用户反馈或直接展开

收到用户的填充反馈后，开始填内容。

**重组规则**：

| 骨架单元 | 重组为 | 说明 |
|---|---|---|
| 核心观点 | `[!NOTE]` Callout | 融入用户的判断 |
| 对比关系 | 表格（表头 emoji） | 对号入座填表 |
| 风险/坑 | `[!WARNING]` Callout | 知识卡片中提取的风险信息 |
| 实用技巧 | `[!TIP]` Callout | |
| 案例/场景 | 列表或简短描述 | |
| 参考资料 | `[!quote]` 或尾部引用列表 | |

**格式约束**（严格遵循 style_guide）：
- 章节编号 `# N.`，二级 `## N.M`
- 每 2-3 个信息单元一个 Callout
- 三行以上内容用表格
- 关键术语首次出现加粗
- 用"我"下判断（用户已提供的判断直接引用）
- 不要 `---` 分割线
- 不要"本文将"、"综上所述"、"双刃剑"等禁词

**诚实约束**：没实操过的不写得像用过一样，不确定的数据不写确定数字。

### 步骤五：交叉引用注入 (Cross-links)

1. 验证正文中所有 `[[链接]]` 指向真实存在的文件
2. 补充遗漏的交叉引用——骨架中明确标注过的已有卡片链接如有遗漏则补全，不主动扩展至骨架标注范围外
3. 填充 Frontmatter 中的 `interlinks` 字段（列出所有引用）
4. 填充 `source_cards` 字段（列出所有参考的知识卡片）

### 步骤六：卡片审计 (Audit)

执行 `style_guide.md` 第十三节「自查清单」的全部四层审计（13.1 格式层、13.2 语言层、13.3 人格层、13.4 诚实层）以及 13.5 不通过时的处理策略。

**前置动作**：先跑第十三节每条 `可执行检查` 的机械扫描（禁词正则、态度词扫描、信息边界标志词检查、"说人话"泛滥检查、假性亲和力检查、食物类比检查），再对通过机械检查的项做主观判断。

**禁止**：把"缺一个 callout"、"差一个过渡句"这类局部问题当成人格/诚实的失败而推倒整篇。

### 步骤七：写入卡片 (Write)

1. 写入 `./06-AI共建/[项目名称]/输出卡片/LOOP-xxx 「Emoji 标题」.md`
2. ID 编号：检查同目录现有卡片，按序递增
3. 反馈：告知用户文件路径，报告审计结果
4. 提示迁移路径：`LOOP-xxx` 是工作区 ID，如需移至 `01-万象火锅/` 正式入库，需分配 KNOW-/TECH- 前缀 ID 并修改 `nodeType` 为 `知识`/`教程`/`技术`

## 4. 交互示例

> User: 骨架有了，帮我出最终卡片。
> Agent (加载约束 + 骨架 + 知识卡片): 好的。我先出框架，你确认关键判断后我再填充。
>
> 这是卡片框架：
> - 4 个一级章节：定义 → 六原语 → 工具对比 → 参考
> - 3 张表格（四种循环对比、六原语风险、工具支持矩阵），表头已设
> - 5 处标注了 `<!-- TODO: 你的判断 -->`，请填入你的观点
>
> User: 框架 OK。我在 3 处填了我的判断，另外删掉了"应用路径"那一节。
> Agent (填充): 收到。基于你的判断，我填完了整张卡片。审计结果：结构 ✓、格式 ✓、交叉引用（补充了 2 个遗漏链接）✓、人格 ✓。卡片已写入 `输出卡片/LOOP-001 「🔄Loop Engineering」.md`。
